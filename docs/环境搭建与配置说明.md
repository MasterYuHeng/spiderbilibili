## 7. 数据库与缓存服务配置

## 7.1 使用 Docker 启动 PostgreSQL 与 Redis

推荐在项目根目录创建 `docker-compose.yml`，示例：

```yaml
version: "3.9"

services:
  postgres:
    image: postgres:16
    container_name: spiderbilibili-postgres
    restart: unless-stopped
    environment:
      POSTGRES_DB: spider_bilibili
      POSTGRES_USER: spider_user
      POSTGRES_PASSWORD: spider_pass
    ports:
      - "5434:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data

  redis:
    image: redis:7
    container_name: spiderbilibili-redis
    restart: unless-stopped
    ports:
      - "6381:6379"
    command: redis-server --appendonly yes
    volumes:
      - redis_data:/data

volumes:
  postgres_data:
  redis_data:
```

启动命令：

```powershell
docker compose up -d
```

查看状态：

```powershell
docker compose ps
```

## 7.2 PostgreSQL 初始化建议

建议数据库信息如下：
- 数据库名：`spider_bilibili`
- 用户名：`spider_user`
- 密码：`spider_pass`
- 当前仓库默认映射端口：`5434`

连接串示例：

```text
postgresql+psycopg://spider_user:spider_pass@127.0.0.1:5434/spider_bilibili
```

## 7.3 Redis 配置建议

Redis 地址示例：

```text
redis://127.0.0.1:6381/0
```

Celery Broker 建议：

```text
redis://127.0.0.1:6381/1
```

Celery Result Backend 建议：

```text
redis://127.0.0.1:6381/2
```

## 9. AI 服务配置说明

### 9.1 标准 OpenAI 协议
如果使用兼容 OpenAI 协议的服务，只需要配置：
- `OPENAI_API_KEY`
- `OPENAI_BASE_URL`
- `OPENAI_MODEL`

### 9.2 模型建议

第一版推荐两类模型：
- 低成本摘要模型：用于单视频总结与主题提取
- 更强分析模型：用于全量主题归并和复杂分类

建议策略：
- 视频级摘要走低成本模型
- 全量分析走更强模型
- 严格限制 token 和批量大小，避免成本失控

## 10. 爬虫环境与代理配置

## 10.1 本地 IP + 休眠策略

第一版建议默认配置：
- 并发数：`1`
- 请求间隔：`1.5s - 5s`
- 分页间隔：`3s - 8s`
- 错误重试：最多 `3` 次
- 失败后指数退避

## 10.2 高质量代理环境变量

如果后续接入高质量代理，可扩展如下配置：

```env
CRAWLER_ENABLE_PROXY=true
PROXY_PROVIDER=custom
HTTP_PROXY=http://username:password@proxy-host:port
HTTPS_PROXY=http://username:password@proxy-host:port
```

说明：
- 推荐稳定的住宅代理或高质量静态代理
- 不建议使用免费代理或不明来源代理池

## 10.3 浏览器自动化注意事项

Playwright 主要作为兜底方案：
- 搜索结果页字段缺失时再启用
- 避免所有请求都走浏览器自动化
- 浏览器上下文要控制频率与实例数

## 14. 常见问题

### 14.1 PowerShell 无法执行激活脚本

执行：

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 14.2 Playwright 浏览器安装失败

可尝试：
- 切换网络环境
- 使用管理员权限 PowerShell
- 在 WSL2 中安装

### 14.3 PostgreSQL 端口占用

修改 `docker-compose.yml` 映射端口，例如：

```yaml
ports:
  - "5434:5432"
```

然后同步更新 `DATABASE_URL`。当前仓库已使用 `5434` 作为 PostgreSQL 映射端口，`6381` 作为 Redis 映射端口。

### 14.4 Redis 未安装但想快速开始

优先直接用 Docker：

```powershell
docker run -d --name spiderbilibili-redis -p 6379:6379 redis:7
```
