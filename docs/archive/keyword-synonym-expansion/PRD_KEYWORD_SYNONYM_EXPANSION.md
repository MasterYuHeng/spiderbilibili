# PRD：关键词同义补充功能

> 归档说明：本文件已在 2026-04-14 文档治理后移入归档目录，保留为专题 PRD 参考。当前项目级主 PRD 以 `../../01-prd.md` 为准，本文件只用于追溯该增强功能的详细设计。

## 1. 文档信息

- 功能名称：关键词同义补充
- 所属模块：任务创建 / 关键词抓取
- 当前状态：需求冻结
- 配套规格：[SPEC_KEYWORD_SYNONYM_EXPANSION_PHASE0.md](./SPEC_KEYWORD_SYNONYM_EXPANSION_PHASE0.md)

## 2. 背景与问题

当前项目的关键词抓取任务以用户输入的单一关键词作为搜索入口，例如“和平精英”。但在 B 站内容生态中，很多视频不会直接使用标准词，而会使用：

- 同义表达
- 常用简称
- 社区流行语
- 口语化代称

例如：

- “和平精英”常被表达为“吃鸡”
- “人工智能”常被表达为“AI”
- “黑神话：悟空”可能被表达为“黑猴”

这会导致当前关键词抓取存在两个核心问题：

- 漏抓大量与主题强相关、但未直接出现原关键词的视频
- 后续主题分析、热门 UP 主分析和报告输出的样本覆盖面不足

因此需要为关键词抓取任务增加“关键词同义补充”能力，在任务执行前先扩展搜索词，再统一进入现有抓取与分析链路。

## 3. 目标

### 3.1 产品目标

在不改变现有任务主链路结构的前提下，提升关键词任务的搜索召回能力。

### 3.2 业务目标

- 提升关键词任务的视频候选覆盖率
- 降低因口语、黑话、流行语造成的漏抓
- 保持用户使用成本尽可能低，只新增必要配置

### 3.3 设计原则

- 默认关闭，不打扰原有用户
- 扩词仅作用于抓取阶段
- 后续分析仍是同一个任务，不按同义词拆分
- 用户可控，可选择是否扩词以及扩词数量
- 扩词失败不阻断主任务，而是自动回退到原关键词抓取

## 4. 非目标

本期不包含以下能力：

- 用户手动维护同义词词库
- 为每个同义词生成独立分析结果
- 在主题分析页按原词/同义词分别展示统计
- 多轮自动扩词
- 热榜抓取模式下的同义词扩充

## 5. 目标用户

- 研究 B 站某个内容主题的运营和分析人员
- 跟踪某类话题热度的内容研究者
- 需要尽量覆盖社区口语表达的产品和市场人员

## 6. 用户场景

### 场景 1：游戏口语表达

用户输入“和平精英”，启用关键词同义补充，补充数量选择 1。系统调用大模型生成 1 个高相关同义词，例如“吃鸡”。后续抓取阶段分别对“和平精英”和“吃鸡”执行搜索，并合并去重后进入现有分析流程。

### 场景 2：行业简称表达

用户输入“人工智能”，启用同义补充并选择 2。系统可能生成“AI”“AIGC”。系统使用这 3 个搜索词抓取候选视频，并基于合并后的结果继续原有主题分析和报告生成。

### 场景 3：用户保留原有行为

用户输入关键词但不启用同义补充。系统完全按照现有逻辑运行，不产生行为变化。

## 7. 核心功能

### 7.1 任务创建页新增配置

仅在“关键词抓取”模式下显示以下配置：

- `enable_keyword_synonym_expansion`
- `keyword_synonym_count`

对应交互：

- 开关：启用关键词同义补充
- 数量：补充 1 / 2 / 3 / 5 个

交互规则：

- 默认关闭
- 关闭时数量控件隐藏或禁用
- 仅当 `crawl_mode = keyword` 时展示
- 当关键词为空时，不允许启用

### 7.2 大模型生成同义词

用户启用该功能后，系统在任务正式进入爬取前，先调用大模型生成同义词列表。

生成要求：

- 结果必须优先是 B 站语境下真实常用的表达
- 优先返回简称、口语词、流行语
- 不返回与原词完全重复的词
- 不返回明显句子化表达
- 不返回泛化词、噪声词、弱相关词

示例：

- 原关键词：“和平精英”
- 补充数量：1
- 生成结果：["吃鸡"]
- 最终搜索词：["和平精英", "吃鸡"]

### 7.3 扩词只作用于抓取阶段

扩展后的搜索词只用于视频搜索与候选抓取阶段。

后续链路保持如下口径：

- 仍只创建一个任务
- 仍只生成一套主题分析
- 仍只生成一套热门 UP 主分析
- 仍只生成一套任务报告
- 任务主标题、任务列表展示名称仍使用原始关键词

### 7.4 多关键词抓取与结果合并

系统对“原关键词 + 同义词列表”逐个执行搜索抓取，再统一合并。

合并规则：

- 按 `bvid` 去重
- 同一视频被多个搜索词命中时，保留一条视频记录
- 为每条视频记录命中的搜索词列表和主命中词

## 9. 功能流程

### 9.1 主流程

1. 用户进入任务创建页
2. 输入原始关键词
3. 选择是否启用关键词同义补充
4. 若启用，则选择补充个数
5. 提交任务
6. 后端创建任务，保存原始关键词与扩词配置
7. 任务执行前调用大模型生成同义词
8. 将原始关键词与同义词组成搜索词列表
9. 逐个搜索词执行视频抓取
10. 合并、去重候选视频
11. 进入现有详情抓取、AI 总结、主题聚类、报告链路
12. 任务完成，结果按单任务统一展示

### 9.2 失败回退流程

若大模型扩词失败：

- 记录 warning 日志
- 自动回退为仅使用原始关键词抓取
- 不阻断整条任务链路

## 10. 数据与接口口径

### 10.1 创建任务请求

在现有 `TaskCreateRequest` 基础上新增：

- `enable_keyword_synonym_expansion: boolean | null`
- `keyword_synonym_count: number | null`

约束：

- 仅当 `crawl_mode = keyword` 时允许生效
- `keyword_synonym_count` 固定允许值为 `1 | 2 | 3 | 5`
- 未开启时 `keyword_synonym_count = null`

### 10.2 extra_params 固定结构

以 [SPEC_KEYWORD_SYNONYM_EXPANSION_PHASE0.md](./SPEC_KEYWORD_SYNONYM_EXPANSION_PHASE0.md) 为准。

核心结构：

```json
{
  "task_options": {
    "enable_keyword_synonym_expansion": true,
    "keyword_synonym_count": 2
  },
  "keyword_expansion": {
    "source_keyword": "和平精英",
    "enabled": true,
    "requested_synonym_count": 2,
    "generated_synonyms": ["吃鸡", "鸡友"],
    "expanded_keywords": ["和平精英", "吃鸡", "鸡友"],
    "status": "success",
    "model_name": "gpt-4.1-mini",
    "error_message": null,
    "generated_at": "2026-04-13T12:00:00Z"
  }
}
```

### 10.3 视频结果字段

每条任务视频需要具备以下固定字段：

- `matched_keywords: string[]`
- `primary_matched_keyword: string | null`
- `keyword_match_count: number`

## 11. 质量与算法规则

### 11.1 同义词生成规则

- 贴近 B 站社区语境
- 与原词主题高度相关
- 优先短词、常用词、搜索词
- 去重
- 不包含原词本身
- 不包含空值

### 11.2 清洗规则

- 去除重复词
- 去除与原词完全相同的词
- 去除过长短语
- 去除明显句子化表达
- 去除“视频”“合集”“教程”“官方”等泛词

### 11.3 搜索执行规则

- 原关键词始终保留在搜索词列表中
- 同义词按顺序追加执行
- 多搜索词抓取结果统一合并
- 评分仍围绕原始关键词进行

## 12. 兼容性要求

- 不影响现有未开启扩词的任务
- 不影响热榜抓取模式
- 不改变当前主题分析、UP 主分析、报告接口结构
- 不要求历史任务补录扩词信息

## 13. 风险与应对

### 风险 1：大模型返回低质量词

应对：

- 限制补充数量上限
- 增加清洗规则
- 失败时回退原关键词

### 风险 2：扩词过多导致抓取量膨胀

应对：

- 默认关闭
- 默认推荐数量为 1
- 固定允许值，只开放少量可选项

### 风险 3：用户误解为多任务分析

应对：

- 在创建页、详情页、报告页明确说明：扩词只增强搜索召回，不会拆成多个分析任务

## 14. 成功指标

- 开启扩词任务占比
- 平均候选视频数提升比例
- 最终入选视频数提升比例
- 用户复用率
- 因扩词引发的失败率或超时率变化

## 15. 验收标准

### 15.1 前端

- 在关键词抓取模式下可以看到扩词开关
- 开启后可以选择补充个数
- 热榜模式下不展示该配置
- 提交任务时可正确传递新增字段

### 15.2 后端

- 能接收并校验扩词配置
- 能在任务执行前调用大模型生成同义词
- 扩词失败时自动回退，不阻断主任务
- 能将原关键词与同义词列表用于抓取
- 抓取结果可完成合并去重

### 15.3 链路

给定：

- 原关键词：和平精英
- 启用同义补充：是
- 补充个数：1

期望：

- 任务最终使用“和平精英”“吃鸡”进行抓取
- 任务列表仍显示“和平精英”
- 主题分析、UP 主分析、任务报告仍只生成一套
- 日志、详情、视频结果和导出结果中都能看到扩词信息

## 16. 结论

“关键词同义补充”本质上是对现有关键词抓取任务的一次召回增强。它必须始终保持：

- 用户只创建一个任务
- 系统只输出一套分析结果
- 扩词只影响前置搜索抓取

这样既能解决社区流行语导致的漏抓问题，又不会破坏当前任务、分析、报告链路的稳定性与理解成本。
