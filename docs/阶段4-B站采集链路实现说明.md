# 阶段4：B站采集链路实现说明

## 1. 目标

阶段 4 的目标是把“关键词 -> 搜索候选视频 -> 视频详情 -> 字幕文本 -> 综合评分”这条采集链路真正打通，并接入现有任务系统与 Worker。

本阶段完成后，系统已经具备以下能力：

- 根据关键词抓取 B 站公开视频搜索结果
- 解析候选视频标题、简介、UP 主、发布时间、搜索位次等基础字段
- 根据 `bvid` 拉取视频详情、标签和基础互动数据
- 在公开视频存在字幕时，抓取并整合字幕文本
- 按“相关性 + 热度”给视频打综合分
- 在 HTTP 请求被风控或字段缺失时，使用 Playwright 做浏览器兜底
- 把本轮采集的原始响应按任务维度归档，方便调试和后续回放

## 2. 数据源分析

本项目当前使用的是公开可访问的 Web 接口与公开视频页数据，不依赖登录态写操作。

### 2.1 搜索结果

- 接口：`GET https://api.bilibili.com/x/web-interface/search/type`
- 关键参数：
  - `search_type=video`
  - `keyword=<关键词>`
  - `page=<页码>`
- 主要字段映射：
  - 标题：`title`
  - 简介：`description`
  - 视频 ID：`bvid`
  - UP 主：`author`
  - 发布时间：`pubdate`
  - 搜索结果页链接：`arcurl`
  - 搜索命中列：`hit_columns`
  - 搜索位次：`rank_index` / `rank_offset`
  - 基础指标：`play`、`like`、`favorites`、`review`、`video_review`

### 2.2 视频详情

- 接口：`GET https://api.bilibili.com/x/web-interface/view/detail`
- 关键参数：
  - `bvid=<视频 BVID>`
- 主要字段映射：
  - 标题：`data.View.title`
  - 简介：`data.View.desc`
  - 发布时间：`data.View.pubdate`
  - 视频封面：`data.View.pic`
  - 分 P：`data.View.pages`
  - 作者信息：`data.Card.card`
  - 互动指标：`data.View.stat`
  - 标签：`data.Tags`

### 2.3 字幕元信息与字幕正文

- 接口：`GET https://api.bilibili.com/x/player/wbi/v2`
- 关键参数：
  - `bvid=<视频 BVID>`
  - `cid=<稿件分 P CID>`
- 说明：
  - 字幕元信息位于 `data.subtitle.subtitles`
  - 真正字幕正文需要继续请求 `subtitle_url`
  - 部分视频没有公开字幕，这属于正常情况

### 2.4 WBI 签名

- 当前实现支持 WBI 参数签名能力
- WBI 密钥来自：`GET https://api.bilibili.com/x/web-interface/nav`
- 对需要签名的接口，可通过 `client.get_api_json(..., use_wbi=True)` 启用
- 目前阶段 4 中，字幕相关接口已经为后续签名调用预留了能力

## 4. 请求策略与合规控制

当前实现遵循“轻量 HTTP 为主，浏览器兜底为辅”的策略。

### 4.1 HTTP 客户端能力

- 默认 `User-Agent`
- 超时控制
- 重试机制
- 412 / 429 风控重试
- 5xx 重试
- 随机休眠
- 每分钟频率限制
- 支持代理环境变量

相关配置位于：

- [config.py](E:\code\fullstack\spiderbilibili\backend\app\core\config.py)
- [backend/.env](E:\code\fullstack\spiderbilibili\backend\.env)
- [backend/.env.example](E:\code\fullstack\spiderbilibili\backend\.env.example)

### 4.2 浏览器兜底

当纯 HTTP 请求遇到以下问题时，采集器会尝试使用 Playwright 兜底：

- HTTP 接口被风控拦截
- 部分接口需要页面上下文
- 需要模拟浏览器环境执行 `fetch`

当前浏览器兜底实现为：

- 先打开 B 站 referer 页面
- 再在浏览器上下文里执行 `fetch`
- 只在必要场景启用，不作为默认主通道

## 5. 去重与评分策略

### 5.1 去重

当前按 `bvid` 去重，保留搜索位次更优、指标更完整的候选结果。

### 5.2 综合评分

综合评分 = 相关性分数 + 热度分数

相关性主要考虑：

- 标题是否命中关键词
- 简介是否命中关键词
- 标签是否命中关键词
- 搜索返回的 `hit_columns`

热度主要考虑：

- 播放量
- 点赞量
- 投币量
- 收藏量
- 评论量
- 弹幕量

热度使用对数归一化，避免超大热视频完全淹没中等热视频。

默认权重：

- `relevance_weight = 0.6`
- `heat_weight = 0.4`

该配置已支持从系统配置表 `analysis.scoring_weights` 覆盖。
