# 阶段15-上线前验收说明

## 1. 阶段目标

阶段 15 的目标不是再补一个单点功能，而是把项目收口成“可上线、可验收、可交接”的完整状态。验收范围覆盖：

- 功能验收：任务创建、任务执行、视频结果、AI 摘要、主题统计、结果导出、失败重试
- 数据验收：视频字段完整率、主题可读性、热度统计逻辑、去重结果
- 稳定性验收：日志完整、任务终态可追踪、失败任务可重试、服务重启后可恢复
- 合规性验收：频率限制、休眠窗口、代理策略受控

## 2. 已交付能力

### 2.1 任务级验收报告

后端新增任务级阶段 15 验收报告能力：

- `GET /api/tasks/{task_id}/acceptance`
  - 返回当前任务的阶段 15 验收结果
  - 输出 `functional`、`data`、`stability`、`compliance` 四个分区
  - 每个分区包含多条检查项，字段包括：
    - `code`
    - `title`
    - `status`
    - `message`
    - `actual`
    - `expected`

报告总状态规则：

- 只要任一检查为 `fail`，总状态为 `fail`
- 无 `fail` 但存在 `warn`，总状态为 `warn`
- 全部通过时，总状态为 `pass`

### 2.2 前端验收页面

前端新增：

- `/tasks/{task_id}/acceptance`

页面能力：

- 展示任务级阶段 15 总状态
- 展示通过 / 警告 / 失败项数量
- 按四类验收项分区展示详细检查结果
- 运行中任务自动轮询刷新
- 从任务详情页和侧边栏直接进入验收页

### 2.3 CLI 验收脚本

仓库根目录提供统一验收脚本：

```powershell
.\.venv\Scripts\python.exe .\scripts\run-stage15-acceptance.py
```

如果要追加某个任务的任务级验收报告：

```powershell
.\.venv\Scripts\python.exe .\scripts\run-stage15-acceptance.py --task-id <task-id>
```

可选参数：

- `--format json|markdown`
- `--output <path>`
- `--skip-backend`
- `--skip-frontend`
- `--skip-docker`

### 2.4 失败任务重试

阶段 15 统一要求失败任务具备恢复入口，当前提供：

- API：

```http
POST /api/tasks/{task_id}/retry
```

- 脚本：

```powershell
.\.venv\Scripts\python.exe .\scripts\retry-task.py --task-id <task-id>
```

前端任务列表页与任务详情页也已提供重试入口。

## 4. 验收判定建议

### 4.1 可直接上线

满足以下条件可视为通过：

- 项目级验收脚本返回 `pass`
- 关键任务的任务级验收结果为 `pass`
- 前后端测试、lint、build 均通过
- Docker 生产配置校验通过

### 4.2 可灰度上线

满足以下条件可视为可灰度上线：

- 项目级验收脚本返回 `pass` 或 `warn`
- 任务级验收仅包含 `warn`，无 `fail`
- `warn` 项不涉及合规性和核心数据错误

### 4.3 禁止上线

出现以下任一情况建议阻塞上线：

- 任一验收结果为 `fail`
- 任务级数据去重失败
- 热度 / 综合评分存在明显异常
- 频率限制、休眠窗口或代理策略不合法
