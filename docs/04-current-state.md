## 2. 本轮扩展影响范围

- 影响需求解读：用户现在可以在主题分析中自定义爆发力、内容深度、社区扩散、完播代理分、综合互动率、主题热度指数的权重。
- 影响产品口径：权重修改不是即时覆盖旧结果，而是“编辑 -> 确认 -> 重新分析 -> 输出新结果”的任务级重算流程。
- 影响技术方案：新增任务级分析权重配置存储、权重更新接口、分析快照重算和报告快照重算。
- 影响前端交互：主题分析页新增权重编辑区、恢复默认、二次确认和重新分析入口。
- 不影响核心架构边界：不重跑抓取链路，不改变单任务、一套分析结果、一套报告的主结构。

## 4. 本轮实现结果

本轮已经完成以下实现：

- 后端新增任务级分析权重配置能力，覆盖爆发力、内容深度、社区扩散、完播代理分、综合互动率、主题热度指数。
- 后端新增“确认后重分析”接口，用于保存权重配置并重新生成主题分析快照和任务报告快照。
- 统计分析逻辑已从固定写死公式改为读取任务级权重配置，并将当前公式与权重说明写回分析结果。
- 主题分析页已支持：
  - 查看当前权重公式
  - 编辑各指标分量权重
  - 恢复默认权重
  - 二次确认后重新分析
- 重新分析后的结果会同步影响：
  - 主题分析页中的指标公式与数据说明
  - 主题热度指数相关可视化
  - 任务报告快照
  - AI 分析上下文中的权重说明

本轮直接修复的确定问题：

- `frontend/src/views/TopicAnalysisView.vue` 中权重输入事件的隐式 `any` 已改为不会触发 TypeScript 构建错误的写法。
- 新增的 `TopicAnalysisView` 测试文件已补齐 Element Plus 组件桩和严格类型，避免测试能过但构建失败。
