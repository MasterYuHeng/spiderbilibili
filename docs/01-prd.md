## 2. 产品定位

SpiderBilibili 是一个面向中文内容研究场景的 Bilibili 关键词采集与分析平台，目标是把“任务创建、抓取执行、分析生成、报告交付、上线验收”组织为可重复执行的完整工作流。

## 3. 目标用户

- 需要研究 Bilibili 某个内容主题的运营人员。
- 需要跟踪话题热度和内容结构的研究人员。
- 需要观察热门内容、主题分布和 UP 主表现的产品或市场人员。

## 4. 产品目标

### 4.1 核心目标

- 让用户能用任务制方式稳定采集 Bilibili 内容数据。
- 将采集结果沉淀为可查看、可分析、可导出的结果资产。
- 通过 AI 辅助生成摘要、主题聚类和报告，降低人工整理成本。

### 4.2 体验目标

- 支持从本地快速启动到完整部署的多种使用方式。
- 让非工程用户也能完成关键配置和基础使用。
- 在功能增强时尽量保持已有任务链路和认知负担稳定。

## 5. 核心业务流程

1. 用户创建任务并设置抓取参数。
2. 系统执行搜索抓取或热榜抓取，并保存原始数据与候选结果。
3. 系统继续执行详情抓取、结果分析和统计计算。
4. 用户查看任务详情、视频结果、主题分析、UP 主分析和任务报告。
5. 用户导出结果或进入上线验收视图。

## 6. 功能范围

### 6.1 任务管理

- 创建任务。
- 任务列表与详情。
- 暂停 / 恢复 / 取消 / 重试。
- 删除、恢复、回收站管理。
- 进度查看。

### 6.2 抓取模式

- 关键词抓取。
- 当前热榜抓取。
- 站内范围与分区范围控制。
- 发布时间限制支持预设最近天数和自定义最近 N 天。
- 原始数据归档与抓取过程记录。

### 6.3 分析与结果

- 视频结果列表。
- 主题分析。
- 热门 UP 主分析。
- 任务报告。
- 上线验收。
- 结果导出。

### 6.4 系统设置

- DeepSeek / OpenAI 兼容 AI 配置。
- Bilibili 登录态配置与浏览器导入。
- 健康检查与运行指标。

### 6.5 运行与部署

- Windows 一键启动。
- 手动本地开发。
- Docker 启动基础设施。
- Docker 整套部署。

## 7. 当前重点增强能力

### 7.1 关键词同义补充

当前已确认纳入主产品范围，但不再单独占据项目主文档结构。
该能力的产品要求是：

- 仅在关键词抓取模式下生效。
- 只增强召回，不拆分任务、不拆分报告。
- 失败时回退，不阻断主任务。
- 任务详情、视频结果、报告应能体现搜索口径。

详细冻结规格和阶段性实施 / 回归记录保存在：

- `docs/archive/keyword-synonym-expansion/PRD_KEYWORD_SYNONYM_EXPANSION.md`
- `docs/archive/keyword-synonym-expansion/IMPLEMENTATION_PLAN_KEYWORD_SYNONYM_EXPANSION.md`
- `docs/archive/keyword-synonym-expansion/REGRESSION_PHASE14_KEYWORD_SYNONYM_EXPANSION.md`

### 7.2 主题分析权重自定义

本轮新增的产品能力要求是：

- 用户可以在主题分析里编辑爆发力、内容深度、社区扩散、完播代理分、综合互动率、主题热度指数的权重配置。
- 用户提交修改时必须二次确认，确认后才开始新的分析重算。
- 重算后需要同步更新主题分析页面、任务报告和 AI 解读内容。
- 这项能力默认作用于已有任务的分析阶段，不要求重新抓取搜索结果或视频详情。
- 页面必须能清晰展示当前指标公式和当前使用的权重配置，避免用户误读结果。

## 8. 非目标

当前主 PRD 不把以下内容作为默认范围：

- 多任务编排系统。
- 面向任意站点的通用爬虫平台。
- 完整账号体系与多租户权限系统。
- 独立的数据仓库产品化能力。
- 每个关键词同义词单独生成一套分析结果。

## 9. 非功能要求

- 保持单任务链路的稳定性和可理解性。
- 关键配置不应硬编码在仓库内。
- 本地开发和 Docker 部署都应可用。
- 结果接口、页面和导出结构需要保持一致口径。
- 文档更新必须可追溯，且优先落到 `docs/00-05` 主文档。
- 用户自定义分析权重后，页面展示、热点结论和 AI 报告不能出现不同步的旧快照。

## 10. 成功标准

- 用户可以稳定创建、执行和查看任务。
- 核心结果页和报告页能够支持内容研究场景。
- 本地启动和 Docker 部署路径清晰。
- 当前增强能力与现有链路兼容，且文档能够准确反映当前状态。
- 用户修改分析权重后，能够明确确认、完成重分析，并得到与新权重一致的图表、热点结论和 AI 报告。
