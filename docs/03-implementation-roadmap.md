## 3. 阶段 5 之后的增量扩展口径

阶段 5 完成后，项目仍可继续按“先文档、再实现”的方式做小步增量扩展。

### 3.1 当前增量扩展：主题分析权重自定义

状态：本轮已完成

目标：

- 允许用户自定义爆发力、内容深度、社区扩散、完播代理分、综合互动率、主题热度指数的权重。
- 在用户确认后重新开始分析，重算主题分析快照和报告快照。
- 让新的权重配置同步影响图表、热点判断、推荐结果和 AI 报告。

默认实施边界：

- 不重跑抓取链路。
- 不改变单任务、一套分析结果、一套报告的主结构。
- 权重配置按任务级分析配置持久化。

本轮结果：

- 后端已补齐任务级分析权重配置、校验与“确认后重分析”接口。
- 主题分析页已支持权重编辑、恢复默认、二次确认和重新分析。
- 主题分析快照、主题分析图表、任务报告和 AI 分析上下文已同步消费新的权重口径。
- 已完成后端定向测试、前端定向测试、前端 lint、前端 build 与报告链路回归。

## 4. 当前推荐推进顺序

阶段 5 完成后，后续建议按以下顺序推进：

1. 如果目标是冻结当前阶段成果，整理工作区并进入提交 / 备份。
2. 如果目标是进入发布前最后确认，执行真实环境启动验收和 Docker 验收。
3. 如果目标是继续迭代需求，先更新 `docs/00-04`，再进入新一轮实现。

## 5. 当前不建议的推进方式

- 不再把历史专题 PRD / Spec / Plan 直接放回 `docs/` 根目录。
- 不在没有更新 `docs/00-04` 的情况下直接推进需求变更。
- 不在阶段 5 已收口后继续把“文档治理”当成当前主目标重复执行。
